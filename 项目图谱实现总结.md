# 项目-图谱-节点系统实现总结

## ✅ 已完成的功能

### 1. 数据库结构 (Neon PostgreSQL)

已在 `prisma/schema.prisma` 中定义了完整的三层数据结构：

#### 📁 Project (项目表)
- 存储项目的基本信息（名称、描述）
- 每个项目有唯一的 `id` (cuid)
- 统计信息：`nodeCount`、`edgeCount`
- 时间戳：`createdAt`、`updatedAt`

#### 🗺️ Graph (图谱表)
- 存储图谱的基本信息（名称、描述）
- 每个图谱有唯一的 `id` (cuid)
- 通过 `projectId` 关联到项目
- 支持公开/私有设置 (`isPublic`)
- 统计信息：`nodeCount`、`edgeCount`

#### 📍 Node (节点表)
- 存储节点的详细信息
- 每个节点有唯一的 `id` (cuid)
- 同时关联项目 (`projectId`) 和图谱 (`graphId`)
- 支持 2D/3D 坐标 (`x`, `y`, `z`)
- 支持媒体文件：
  - `imageUrl`: 图片URL
  - `iconUrl`: 图标URL
  - `coverUrl`: 封面/视频URL
- 视觉属性：`color`、`size`、`opacity`

#### 🔗 Edge (边表)
- 存储节点之间的关系
- 每条边有唯一的 `id` (cuid)
- 同时关联项目 (`projectId`) 和图谱 (`graphId`)
- 关联起始节点 (`fromNodeId`) 和目标节点 (`toNodeId`)
- 关系属性：`label`、`weight`、`color`、`style`

### 2. API 接口

#### 项目管理 API
- ✅ `POST /api/projects` - 创建项目
- ✅ `GET /api/projects` - 获取所有项目
- ✅ `GET /api/projects/[id]` - 获取项目详情
- ✅ `DELETE /api/projects/[id]` - 删除项目（级联删除）
- ✅ `GET /api/projects/[id]/graphs` - 获取项目的所有图谱
- ✅ `POST /api/projects/[id]/graphs` - 在项目中创建图谱
- ✅ `GET /api/projects/[id]/nodes` - 获取项目的所有节点
- ✅ `POST /api/projects/[id]/nodes` - 在项目中创建节点
- ✅ `GET /api/projects/[id]/edges` - 获取项目的所有边
- ✅ `POST /api/projects/[id]/edges` - 在项目中创建边

#### 图谱管理 API
- ✅ `GET /api/graphs` - 获取所有图谱（可按项目筛选）
- ✅ `POST /api/graphs` - 创建图谱
- ✅ `GET /api/graphs/[id]` - 获取图谱详情
- ✅ `PATCH /api/graphs/[id]` - 更新图谱信息
- ✅ `DELETE /api/graphs/[id]` - 删除图谱（级联删除）
- ✅ `GET /api/graphs/[id]/nodes` - 获取图谱的所有节点
- ✅ `POST /api/graphs/[id]/nodes` - 在图谱中创建节点
- ✅ `GET /api/graphs/[id]/edges` - 获取图谱的所有边
- ✅ `POST /api/graphs/[id]/edges` - 在图谱中创建边

### 3. 数据关联机制

#### 三层关联结构
```
Project (项目)
  ├── id: "cmkxxx..."
  ├── name: "委屈"
  └── graphs: [
       ├── Graph (图谱A)
       │    ├── id: "cmkyyy..."
       │    ├── projectId: "cmkxxx..."
       │    ├── nodes: [
       │    │    ├── Node (节点1)
       │    │    │    ├── id: "cmkzzz..."
       │    │    │    ├── projectId: "cmkxxx..."
       │    │    │    ├── graphId: "cmkyyy..."
       │    │    │    ├── imageUrl: "https://..."
       │    │    │    └── coverUrl: "https://..."
       │    │    └── Node (节点2)...
       │    │    ]
       │    └── edges: [
       │         ├── Edge (边1)
       │         │    ├── id: "cmkaaa..."
       │         │    ├── projectId: "cmkxxx..."
       │         │    ├── graphId: "cmkyyy..."
       │         │    ├── fromNodeId: "cmkzzz..."
       │         │    └── toNodeId: "cmkbbb..."
       │         └── Edge (边2)...
       │         ]
       └── Graph (图谱B)...
       ]
```

#### 数据隔离保证
1. **项目级隔离**: 每个项目的数据完全独立
2. **图谱级隔离**: 每个图谱的节点和边只属于该图谱
3. **双重关联**: 节点和边同时关联项目和图谱，确保数据一致性
4. **级联删除**: 删除项目会自动删除所有图谱、节点、边和媒体文件

### 4. 媒体文件存储

节点支持三种媒体URL：
- `imageUrl`: 节点的主图片
- `iconUrl`: 节点的图标
- `coverUrl`: 节点的封面或视频

媒体文件存储在 Vercel Blob，数据库只保存URL地址。

## 🎯 使用示例

### 创建完整的项目结构

```javascript
// 1. 创建项目 "委屈"
const projectRes = await fetch('/api/projects', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    name: '委屈',
    description: '我的第一个项目'
  })
})
const { project } = await projectRes.json()
console.log('项目ID:', project.id)

// 2. 在项目中创建图谱
const graphRes = await fetch(`/api/projects/${project.id}/graphs`, {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    name: '知识图谱A',
    description: '核心知识图谱'
  })
})
const { graph } = await graphRes.json()
console.log('图谱ID:', graph.id)

// 3. 在图谱中创建节点
const node1Res = await fetch(`/api/graphs/${graph.id}/nodes`, {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    name: '核心概念1',
    type: 'concept',
    description: '这是第一个核心概念',
    x: 0, y: 0, z: 0,
    color: '#FF6B6B',
    imageUrl: 'https://blob.vercel-storage.com/xxx.jpg'
  })
})
const { node: node1 } = await node1Res.json()

const node2Res = await fetch(`/api/graphs/${graph.id}/nodes`, {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    name: '核心概念2',
    type: 'concept',
    x: 5, y: 0, z: 0,
    color: '#4ECDC4'
  })
})
const { node: node2 } = await node2Res.json()

// 4. 创建节点之间的关系
const edgeRes = await fetch(`/api/graphs/${graph.id}/edges`, {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    fromNodeId: node1.id,
    toNodeId: node2.id,
    label: 'RELATES_TO',
    weight: 1.5
  })
})
```

### 在2D/3D视图中显示项目数据

```javascript
// 获取当前项目的所有节点
const nodesRes = await fetch(`/api/projects/${currentProjectId}/nodes`)
const { nodes } = await nodesRes.json()

// 获取当前项目的所有边
const edgesRes = await fetch(`/api/projects/${currentProjectId}/edges`)
const { edges } = await edgesRes.json()

// 在2D视图中渲染
nodes.forEach(node => {
  renderNode2D(node.x, node.y, node.color, node.size, node.name)
  if (node.imageUrl) {
    displayImage(node.imageUrl)
  }
})

// 在3D视图中渲染
nodes.forEach(node => {
  renderNode3D(node.x, node.y, node.z, node.color, node.size, node.name)
  if (node.imageUrl) {
    displayImage(node.imageUrl)
  }
})

// 渲染边
edges.forEach(edge => {
  const fromNode = nodes.find(n => n.id === edge.fromNodeId)
  const toNode = nodes.find(n => n.id === edge.toNodeId)
  if (fromNode && toNode) {
    renderEdge(fromNode, toNode, edge.color, edge.style)
  }
})
```

### 按图谱筛选显示

```javascript
// 获取特定图谱的节点和边
const graphNodesRes = await fetch(`/api/graphs/${currentGraphId}/nodes`)
const { nodes } = await graphNodesRes.json()

const graphEdgesRes = await fetch(`/api/graphs/${currentGraphId}/edges`)
const { edges } = await graphEdgesRes.json()

// 只显示当前图谱的数据
renderGraph(nodes, edges)
```
