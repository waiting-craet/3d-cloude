# 图谱切换功能使用指南

## 🎯 功能说明

现在你可以在一个项目下创建多个知识图谱，每个图谱的数据完全独立。通过顶部导航栏的"现有图谱"下拉菜单，可以轻松切换不同的图谱。

## 🚀 快速开始

### 1. 清理旧数据（首次使用）

如果你之前使用过旧版本，建议先清理缓存：

**方法1: 访问清理页面**
```
http://localhost:3000/clear-cache
```

**方法2: 手动清理**
在浏览器控制台（F12）中执行：
```javascript
localStorage.clear()
location.reload()
```

### 2. 创建测试数据

运行测试脚本创建示例数据：
```bash
npx tsx scripts/create-test-data.ts
```

这会创建：
- 📁 测试项目
  - 🗺️ 知识图谱A（5个节点，5条边）
  - 🗺️ 知识图谱B（3个节点，2条边）

### 3. 查看图谱

1. 打开网站：http://localhost:3000
2. 点击顶部的"现有图谱"按钮
3. 点击"测试项目"展开
4. 选择"知识图谱A"或"知识图谱B"
5. 3D视图会立即显示对应的节点和边

## 📋 功能详解

### 图谱选择器

```
┌─────────────────────────────────────┐
│  现有图谱  ▼                        │
└─────────────────────────────────────┘
         │
         ▼
┌─────────────────────────────────────┐
│ ▶ 📁 测试项目                       │
│   └─ 2 个图谱                       │
├─────────────────────────────────────┤
│   ✓ 知识图谱A                       │
│     5 节点 · 5 关系                 │
├─────────────────────────────────────┤
│     知识图谱B                       │
│     3 节点 · 2 关系                 │
└─────────────────────────────────────┘
```

### 切换图谱

1. **点击项目名称** - 展开/收起图谱列表
2. **点击图谱名称** - 切换到该图谱
3. **当前图谱** - 显示蓝色勾选标记 ✓
4. **实时统计** - 显示节点数和关系数

### 数据隔离

每个图谱的数据完全独立：

```
图谱A                    图谱B
├─ 节点1                ├─ 节点X
├─ 节点2                ├─ 节点Y
├─ 节点3                └─ 节点Z
└─ 边1-2, 2-3           └─ 边X-Y, Y-Z
```

- 在图谱A中添加节点 → 只在图谱A中显示
- 切换到图谱B → 看不到图谱A的节点
- 切换回图谱A → 之前添加的节点仍然存在

## 🎨 创建新图谱

### 前提条件
需要管理员权限（点击"登录"按钮）

### 创建步骤

1. **登录管理员账号**
   - 点击右上角"登录"
   - 输入管理员凭证

2. **点击"新建图谱"按钮**
   - 出现在登录后的导航栏

3. **选择创建方式**
   - **新建项目**: 创建新项目和第一个图谱
   - **添加到现有项目**: 在已有项目下添加新图谱

4. **填写信息**
   - 项目名称（如果是新项目）
   - 图谱名称

5. **开始使用**
   - 自动切换到新创建的图谱
   - 可以开始添加节点和关系

## 🔧 常见问题

### Q1: 选择图谱后没有显示节点？

**解决方法：**
1. 检查控制台是否有错误
2. 确认图谱中确实有节点（查看统计数字）
3. 尝试刷新页面
4. 清理缓存后重试

### Q2: 看到404错误？

如果控制台显示类似错误：
```
GET /api/graphs/graph-1768294373820/nodes 404
```

这说明使用了旧的本地ID，需要清理缓存：
```
访问: http://localhost:3000/clear-cache
```

### Q3: 切换图谱很慢？

这是正常的，因为需要从数据库加载数据。优化建议：
- 减少单个图谱的节点数量
- 使用多个小图谱而不是一个大图谱
- 检查网络连接

### Q4: 新建的节点在其他图谱中也显示？

这不应该发生。如果出现，说明数据关联有问题：
1. 确认选择了正确的图谱
2. 检查控制台的API调用
3. 刷新页面重新加载

## 📊 数据管理

### 查看项目统计

在"现有图谱"下拉菜单中可以看到：
- 项目名称
- 图谱数量
- 每个图谱的节点数和关系数

### 数据持久化

所有数据都存储在Neon数据库中：
- ✅ 刷新页面不会丢失数据
- ✅ 关闭浏览器后数据仍然存在
- ✅ 可以在不同设备上访问相同的数据

### 删除数据

目前需要通过API删除：
```bash
# 删除图谱
curl -X DELETE http://localhost:3000/api/graphs/{graphId}

# 删除项目（会删除所有图谱）
curl -X DELETE http://localhost:3000/api/projects/{projectId}
```

## 🎯 最佳实践

### 1. 项目组织

```
项目：机器学习
  ├─ 图谱：基础概念
  ├─ 图谱：算法分类
  └─ 图谱：应用场景

项目：前端开发
  ├─ 图谱：React生态
  ├─ 图谱：Vue生态
  └─ 图谱：构建工具
```

### 2. 命名规范

- **项目名称**: 简短明确，如"机器学习"、"前端开发"
- **图谱名称**: 描述性强，如"基础概念"、"算法分类"

### 3. 节点管理

- 每个图谱保持适当的节点数量（建议50个以内）
- 使用有意义的节点名称
- 合理使用颜色区分不同类型的节点

### 4. 关系管理

- 使用清晰的关系标签（如"RELATES_TO"、"PART_OF"）
- 避免创建过多的交叉关系
- 保持图谱结构清晰

## 🔍 调试技巧

### 查看加载日志

打开浏览器控制台（F12），切换图谱时会看到：
```
正在加载图谱数据: cmkci0cvk000r98y75y52pwht
加载的节点数: 5
加载的边数: 5
```

### 检查API调用

在Network标签中查看：
- `GET /api/projects` - 加载项目列表
- `GET /api/projects/{id}/graphs` - 加载图谱列表
- `GET /api/graphs/{id}/nodes` - 加载节点
- `GET /api/graphs/{id}/edges` - 加载边

### 验证数据

使用API直接查询：
```bash
# 查看所有项目
curl http://localhost:3000/api/projects

# 查看项目的图谱
curl http://localhost:3000/api/projects/{projectId}/graphs

# 查看图谱的节点
curl http://localhost:3000/api/graphs/{graphId}/nodes
```
