# 知识图谱导入算法指南

## 概述

本系统实现了一套完善的文件导入和图谱生成算法，支持Excel、CSV、JSON三种格式的文件，能够智能解析数据并生成2D或3D知识图谱。

## 核心功能

### 1. 多格式文件解析

#### Excel文件解析
支持两种Excel格式：

**格式1：独立工作表**
- `Nodes` 工作表：包含节点信息
  - 必需列：`id`, `label`
  - 可选列：`description`, `x`, `y`, `z`（3D）, `color`, `size`, `shape`
- `Edges` 工作表：包含边信息
  - 必需列：`source`, `target`
  - 可选列：`label`, `relationship`

**格式2：单一工作表**
- 只包含边信息：`source`, `target`, `label`
- 系统自动从边数据中提取节点

#### CSV文件解析
支持两种CSV格式：

**格式1：完整节点数据**
```csv
id,label,description,x,y,z
node1,节点A,描述,100,100,0
```

**格式2：边数据格式**
```csv
source,target,label
节点A,节点B,关系1
```

特性：
- 智能识别列名（不区分大小写）
- 支持多种列名变体（source/from/src, target/to/dest等）
- 正确处理CSV中的引号和逗号

#### JSON文件解析
支持多种JSON格式：

**格式1：标准格式**
```json
{
  "nodes": [
    { "id": "node1", "label": "节点A", "x": 100, "y": 100 }
  ],
  "edges": [
    { "source": "node1", "target": "node2", "label": "关系" }
  ]
}
```

**格式2：Cytoscape格式**
```json
{
  "elements": {
    "nodes": [...],
    "edges": [...]
  }
}
```

**格式3：数组格式**
```json
[
  { "source": "A", "target": "B", "label": "关系1" },
  { "source": "B", "target": "C", "label": "关系2" }
]
```

### 2. 智能布局生成算法

当导入的数据缺少节点坐标时，系统会自动使用力导向布局算法生成合理的节点位置。

#### 力导向布局算法

**算法原理：**
1. **初始化**：将节点均匀分布在圆形（2D）或球形（3D）上
2. **斥力计算**：所有节点对之间产生斥力，防止节点重叠
   - 斥力公式：`F_repulsion = k² / distance`
3. **引力计算**：有边连接的节点之间产生引力，保持连接紧密
   - 引力公式：`F_attraction = distance² / k`
4. **迭代优化**：通过多次迭代调整节点位置，逐步达到平衡状态
5. **温度衰减**：随着迭代进行，节点移动幅度逐渐减小

**算法参数：**
- 迭代次数：50次
- 理想距离k：根据节点数量自动计算
- 初始温度：100
- 温度衰减：线性衰减

**算法优势：**
- 自动避免节点重叠
- 保持连接节点的接近性
- 生成美观、易读的图谱布局
- 支持2D和3D布局

### 3. 数据验证和错误处理

#### 文件验证
- 检查文件格式是否正确
- 验证必需字段是否存在
- 处理空行和无效数据

#### 节点验证
- 自动去重（基于label或id）
- 为缺失的必需字段提供默认值
- 验证坐标数据的有效性

#### 边验证
- 检查source和target节点是否存在
- 跳过无效的边（节点不存在）
- 记录警告信息

### 4. 数据库集成

#### 批量创建
- 使用Prisma ORM进行数据库操作
- 为每个节点和边创建数据库记录
- 自动关联到指定的项目和图谱

#### 关系映射
- 创建节点label/id到数据库id的映射
- 确保边的source和target正确关联
- 更新图谱的节点和边计数

## 使用流程

1. **选择项目和图谱**：在导入页面选择目标项目和图谱
2. **选择图谱类型**：选择2D或3D图谱类型
3. **上传文件**：上传Excel、CSV或JSON文件
4. **自动解析**：系统自动识别文件格式并解析数据
5. **生成布局**：为缺少坐标的节点自动生成布局
6. **创建图谱**：将数据保存到数据库并生成可视化图谱

## 技术实现

### 核心文件
- `lib/services/graph-import.ts`：文件解析和布局生成服务
- `app/api/import/route.ts`：导入API端点
- `app/api/templates/route.ts`：模板生成API

### 依赖库
- `xlsx`：Excel文件解析
- `@prisma/client`：数据库操作

### 算法复杂度
- 文件解析：O(n)，n为数据行数
- 力导向布局：O(iterations × n²)，n为节点数
- 数据库插入：O(n + m)，n为节点数，m为边数

## 最佳实践

1. **数据准备**
   - 确保节点id/label唯一
   - 边的source和target必须对应已存在的节点
   - 使用有意义的标签和描述

2. **性能优化**
   - 对于大型图谱（>1000节点），建议提供坐标数据
   - 分批导入大量数据
   - 使用JSON格式以获得最佳性能

3. **布局优化**
   - 对于复杂图谱，可以先导入后手动调整
   - 使用模板文件作为起点
   - 保持节点数量适中（建议<500）

## 错误处理

系统会捕获并报告以下错误：
- 文件格式不正确
- 必需字段缺失
- 数据类型错误
- 节点或边创建失败
- 数据库连接问题

所有错误都会返回详细的错误信息，帮助用户快速定位问题。
