# 节点最小距离更新至 15 单位

## 更新时间
2025-01-18

## 修改内容

将节点之间的最小距离从 **6 单位** 增加到 **15 单位**，这是一个 **2.5 倍**的增加。

## 修改的文件

### 1. `lib/coordinate-converter.ts`
```typescript
// 默认最小距离
const { heightVariation = 5, minNodeDistance = 15 } = config
```

### 2. `app/api/graphs/[id]/sync/route.ts`
```typescript
{
  heightVariation: 5,
  minNodeDistance: 15,  // 从 6 增加到 15
}
```

### 3. `app/api/convert/route.ts`
```typescript
{
  heightVariation: 5,
  minNodeDistance: 15,  // 从 6 增加到 15
}
```

## 效果

### 间距对比

| 版本 | 最小距离 | 说明 |
|------|---------|------|
| 初始版本 | 2 单位 | 太拥挤 |
| 第一次优化 | 4 单位 | 仍然较近 |
| 第二次优化 | 6 单位 | 有所改善 |
| **当前版本** | **15 单位** | **非常宽松** ⭐ |

### 视觉效果

- ✅ 节点之间有**非常大**的空间
- ✅ 即使有 100+ 个节点也不会感到拥挤
- ✅ 每个节点都清晰可见，易于选择
- ✅ 标签不会重叠
- ✅ 整体视觉效果更加专业和清晰

## 使用说明

### 查看新间距效果

1. **刷新浏览器**页面
2. 如果有现有数据，需要**重新同步**或**重新转换**
3. 新创建的图谱会自动使用新的间距

### 重新同步现有图谱

在 2D 工作流页面：
1. 点击"保存并转换为3D"按钮
2. 或点击"同步到3D"按钮
3. 系统会使用新的 15 单位最小距离重新计算节点位置

      // 如果距离 < 15，推开它们

### 性能影响

- 对于小型图谱（< 50 节点）：几乎无影响
- 对于中型图谱（50-100 节点）：轻微增加（< 100ms）
- 对于大型图谱（100+ 节点）：可能增加 100-200ms

这个性能开销是完全可以接受的，因为换来的是更好的视觉效果。

## 后续建议

如果 15 单位仍然不够：
1. 可以继续增加到 20 或 25 单位
2. 可以调整智能间距因子（增加倍数）
3. 可以实现用户自定义间距的配置选项

如果 15 单位太大：
1. 可以减少到 10 或 12 单位
2. 可以根据节点数量动态调整最小距离
