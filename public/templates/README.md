# 知识图谱导入模板说明

## 概述

本目录包含用于导入知识图谱数据的模板文件。所有模板都遵循统一的数据结构，确保与导入算法完全兼容。

## 模板文件列表

- `2d-graph-template.csv` - 2D图谱CSV模板（边数据格式）
- `2d-graph-template.json` - 2D图谱JSON模板（完整格式）
- `3d-graph-template.csv` - 3D图谱CSV模板（边数据格式）
- `3d-graph-template.json` - 3D图谱JSON模板（完整格式）
- Excel模板通过API动态生成

## 数据结构说明

### 节点（Nodes）数据结构

#### 必需字段
- `id` (string): 节点唯一标识符
- `label` (string): 节点显示名称

#### 可选字段
- `description` (string): 节点描述信息
- `x` (number): X轴坐标（如果不提供，系统会自动生成）
- `y` (number): Y轴坐标（如果不提供，系统会自动生成）
- `z` (number): Z轴坐标（仅3D图谱，如果不提供，系统会自动生成）
- `color` (string): 节点颜色（十六进制格式，如 #00bfa5）
- `size` (number): 节点大小（默认为1）
- `shape` (string): 节点形状（默认为sphere）

### 边（Edges）数据结构

#### 必需字段
- `source` (string): 源节点的id或label
- `target` (string): 目标节点的id或label

#### 可选字段
- `label` (string): 边的标签/关系名称

## CSV 模板格式

### 格式1：边数据格式（推荐用于简单图谱）

CSV文件包含边信息，系统会自动提取节点：

```csv
source,target,label
节点A,节点B,关系1
节点A,节点C,关系2
节点B,节点D,关系3
```

**特点**：
- 简单易用，适合快速创建图谱
- 系统自动从边数据中提取节点
- 节点坐标由系统自动生成

### 格式2：完整节点数据格式

CSV文件包含完整的节点信息：

**2D图谱：**
```csv
id,label,description,x,y,color,size,shape
node1,节点A,描述,100,100,#00bfa5,1,sphere
node2,节点B,描述,300,100,#ff6b6b,1.2,sphere
```

**3D图谱：**
```csv
id,label,description,x,y,z,color,size,shape
node1,中心节点,描述,0,0,0,#00bfa5,1.5,sphere
node2,技术节点,描述,100,0,0,#ff6b6b,1,sphere
```

**特点**：
- 完全控制节点属性
- 可以指定精确的坐标位置
- 适合复杂图谱或需要精确布局的场景

## JSON 模板格式

JSON格式提供最完整的数据结构：

### 2D图谱JSON格式
```json
{
  "nodes": [
    {
      "id": "node1",
      "label": "节点A",
      "description": "这是节点A的描述",
      "x": 100,
      "y": 100,
      "color": "#00bfa5",
      "size": 1,
      "shape": "sphere"
    }
  ],
  "edges": [
    {
      "source": "node1",
      "target": "node2",
      "label": "关系1"
    }
  ]
}
```

### 3D图谱JSON格式
```json
{
  "nodes": [
    {
      "id": "node1",
      "label": "中心节点",
      "description": "这是中心节点的描述",
      "x": 0,
      "y": 0,
      "z": 0,
      "color": "#00bfa5",
      "size": 1.5,
      "shape": "sphere"
    }
  ],
  "edges": [
    {
      "source": "node1",
      "target": "node2",
      "label": "技术关系"
    }
  ]
}
```

**特点**：
- 结构清晰，易于编辑
- 支持所有可选字段
- 适合程序化生成数据

## Excel 模板格式

Excel文件包含两个工作表：

### Sheet 1: Nodes（节点）

**2D图谱节点表格：**
| id | label | description | x | y | color | size | shape |
|----|-------|-------------|---|---|-------|------|-------|
| node1 | 节点A | 描述 | 100 | 100 | #00bfa5 | 1 | sphere |
| node2 | 节点B | 描述 | 300 | 100 | #ff6b6b | 1.2 | sphere |

**3D图谱节点表格：**
| id | label | description | x | y | z | color | size | shape |
|----|-------|-------------|---|---|---|-------|------|-------|
| node1 | 中心节点 | 描述 | 0 | 0 | 0 | #00bfa5 | 1.5 | sphere |
| node2 | 技术节点 | 描述 | 100 | 0 | 0 | #ff6b6b | 1 | sphere |

### Sheet 2: Edges（边）
| source | target | label |
|--------|--------|-------|
| node1 | node2 | 关系1 |
| node2 | node3 | 关系2 |

**特点**：
- 适合非技术用户
- 可以使用Excel的编辑功能
- 支持批量数据处理

## 使用说明

1. **选择模板格式**：根据需求选择CSV、JSON或Excel格式
2. **选择图谱类型**：在导入页面选择2D或3D
3. **下载对应模板**：点击模板按钮下载
4. **填写数据**：
   - 保持表头不变
   - 填写节点和边数据
   - 可选字段可以留空（系统会使用默认值）
5. **上传导入**：在导入页面上传填写好的文件

## 注意事项

1. **节点ID唯一性**：确保每个节点的id在图谱中是唯一的
2. **边的引用**：边的source和target必须对应已存在的节点id或label
3. **坐标可选**：如果不提供坐标，系统会使用力导向算法自动生成布局
4. **颜色格式**：颜色必须使用十六进制格式（如 #00bfa5）
5. **数值类型**：x, y, z, size必须是数字类型
6. **文件编码**：CSV文件建议使用UTF-8编码以支持中文

## 高级用法

### 自动布局
如果只提供边数据（不提供节点坐标），系统会：
1. 自动提取所有节点
2. 使用力导向算法生成布局
3. 确保节点不重叠且连接紧密

### 混合模式
可以为部分节点提供坐标，其他节点由系统自动布局：
- 提供坐标的节点会固定在指定位置
- 未提供坐标的节点会自动生成位置

### 批量导入
对于大量数据：
1. 建议使用JSON格式（性能最佳）
2. 可以分批导入到不同的图谱
3. 使用程序化方式生成模板数据
