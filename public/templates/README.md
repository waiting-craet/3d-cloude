# 知识图谱导入模板说明

## 🎯 简化设计理念

**新版模板采用极简设计**,用户只需填写最核心的信息:
- ✅ **节点名称** (必填)
- ✅ **节点描述** (可选)
- ✅ **节点关系** (必填)

**系统自动处理**:
- 🎨 节点颜色 (自动分配美观配色)
- 📍 3D坐标 (智能布局算法)
- 📏 节点大小 (根据连接数自动调整)
- 🔷 节点形状 (统一球形)
- 🆔 节点ID (自动生成唯一标识)

## 📁 模板文件列表

- `3d-graph-template.csv` - **CSV模板** (最简单,推荐)
- `3d-graph-template.json` - **JSON模板** (结构化)
- Excel模板 - **通过API动态生成** (适合批量编辑)

## 🚀 快速开始

### 方式1: CSV模板 (推荐 - 最简单)

只需填写三列:

```csv
source,target,label
产品经理,需求文档,编写
需求文档,开发团队,交付
开发团队,代码,编写
代码,测试团队,交付
测试团队,产品,发布
```

**就这么简单!** 系统会自动:
- 提取所有节点 (产品经理、需求文档、开发团队等)
- 生成3D坐标布局
- 分配美观的颜色
- 设置合适的大小

### 方式2: JSON模板 (结构化)

```json
{
  "nodes": [
    {
      "label": "产品经理",
      "description": "负责产品规划"
    },
    {
      "label": "开发团队",
      "description": "负责代码实现"
    }
  ],
  "edges": [
    {
      "source": "产品经理",
      "target": "开发团队",
      "label": "需求传递"
    }
  ]
}
```

### 方式3: Excel模板

**Sheet 1: Nodes (节点表)**
| label | description |
|-------|-------------|
| 产品经理 | 负责产品规划 |
| 开发团队 | 负责代码实现 |
| 测试团队 | 负责质量保证 |

**Sheet 2: Edges (关系表)**
| source | target | label |
|--------|--------|-------|
| 产品经理 | 开发团队 | 需求传递 |
| 开发团队 | 测试团队 | 代码交付 |

## 📋 数据结构说明

### 节点 (Nodes)

#### 必填字段
- `label` (string): 节点名称
  - 示例: "产品经理", "需求文档", "开发团队"
  - 用作节点的显示名称和唯一标识

#### 可选字段
- `description` (string): 节点描述
  - 示例: "负责产品规划和需求分析"
  - 可以留空,系统会使用默认描述

### 边 (Edges)

#### 必填字段
- `source` (string): 起始节点名称
  - 必须对应nodes中的label
- `target` (string): 目标节点名称
  - 必须对应nodes中的label

#### 可选字段
- `label` (string): 关系名称
  - 示例: "编写", "审核", "发布"
  - 可以留空

## 🎨 自动生成功能

### 1. 智能配色系统
系统会自动为节点分配美观的颜色:
- 🔵 青绿色 (#00bfa5)
- 🔴 红色 (#ff6b6b)
- 🟢 蓝绿色 (#4ecdc4)
- 🟡 黄色 (#ffe66d)
- 🟣 淡紫色 (#c7ceea)
- 🌸 粉红色 (#ff8b94)
- 🌿 浅绿色 (#a8e6cf)

颜色会根据节点在图谱中的位置自动分配,确保视觉效果美观。

### 2. 智能3D布局算法
系统使用先进的力导向布局算法:
- ✅ 自动计算最优节点位置
- ✅ 确保节点不重叠
- ✅ 保持合理的节点间距
- ✅ 突出重要节点(连接数多的节点)
- ✅ 创建层次感和深度感

### 3. 自动大小调整
节点大小根据重要性自动调整:
- 连接数多的节点 → 更大
- 连接数少的节点 → 较小
- 范围: 0.8 - 1.5

### 4. 自动ID生成
系统会为每个节点自动生成唯一ID:
- 基于节点名称生成
- 确保全局唯一性
- 用户无需关心

## 💡 使用技巧

### 技巧1: 最简CSV格式
如果你只想快速创建图谱,只需要一个CSV文件:

```csv
source,target,label
A,B,关系1
B,C,关系2
C,D,关系3
```

系统会自动提取A、B、C、D作为节点!

### 技巧2: 添加节点描述
如果需要更详细的信息,使用JSON或Excel格式:

**JSON:**
```json
{
  "nodes": [
    {"label": "A", "description": "这是A的详细描述"},
    {"label": "B", "description": "这是B的详细描述"}
  ],
  "edges": [
    {"source": "A", "target": "B", "label": "关系"}
  ]
}
```

### 技巧3: 批量导入
对于大量数据:
1. 使用Excel编辑 (支持公式和批量操作)
2. 或使用程序生成JSON文件
3. 系统会自动处理所有布局和样式

### 技巧4: 中文支持
完全支持中文节点名称和关系:
```csv
source,target,label
产品经理,需求文档,编写
需求文档,开发团队,交付
开发团队,测试团队,协作
```

## ⚠️ 注意事项

### 1. 节点名称唯一性
- 每个节点的`label`必须唯一
- 如果有重复,系统会自动合并为同一个节点

### 2. 边的引用
- `source`和`target`必须对应已存在的节点名称
- 大小写敏感: "节点A" ≠ "节点a"

### 3. 文件编码
- CSV文件请使用 **UTF-8编码** 以支持中文
- Excel文件自动处理编码

### 4. 数据量建议
- 小型图谱: < 50个节点 (推荐CSV)
- 中型图谱: 50-200个节点 (推荐Excel)
- 大型图谱: > 200个节点 (推荐JSON + 程序生成)

## 🔧 高级功能

### 混合模式 (可选)
如果你确实需要精确控制某些属性,仍然可以在JSON中指定:

```json
{
  "nodes": [
    {
      "label": "重要节点",
      "description": "这个节点很重要",
      "color": "#ff0000",
      "size": 2.0
    },
    {
      "label": "普通节点"
    }
  ]
}
```

系统会:
- 使用你指定的颜色和大小 (重要节点)
- 自动生成其他节点的属性 (普通节点)
- 为所有节点生成3D坐标
