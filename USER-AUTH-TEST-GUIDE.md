# 用户认证功能测试指南

## 测试步骤

### 1. 访问应用

打开浏览器访问: http://localhost:3000

### 2. 测试注册功能

1. 点击顶部导航栏的 **"登录"** 按钮
2. 在弹窗中点击 **"注册"** 标签页
3. 输入用户名（例如：`testuser`）
4. 输入密码（至少 6 个字符，例如：`password123`）
5. 点击 **"注册"** 按钮

**预期结果**:
- 注册成功后自动登录
- 弹窗关闭
- 顶部导航栏显示用户名（👤 testuser）
- 显示 "新建图谱" 按钮
- 显示 "登出" 按钮

### 3. 测试登录状态持久化

1. 刷新页面 (F5)

**预期结果**:
- 用户仍然保持登录状态
- 顶部导航栏仍然显示用户名

### 4. 测试登出功能

1. 点击顶部导航栏的 **"登出"** 按钮

**预期结果**:
- 用户信息消失
- 显示 "登录" 按钮
- "新建图谱" 按钮消失

### 5. 测试登录功能

1. 点击 **"登录"** 按钮
2. 输入之前注册的用户名和密码
3. 点击 **"登录"** 按钮

**预期结果**:
- 登录成功
- 弹窗关闭
- 显示用户信息

### 6. 测试错误处理

#### 6.1 注册时用户名已存在

1. 点击 "登录" → "注册"
2. 输入已存在的用户名
3. 点击 "注册"

**预期结果**: 显示错误提示 "用户名已被使用"

#### 6.2 登录时密码错误

1. 点击 "登录"
2. 输入正确的用户名，错误的密码
3. 点击 "登录"

**预期结果**: 显示错误提示 "用户名或密码错误"

#### 6.3 密码过短

1. 点击 "登录" → "注册"
2. 输入用户名和少于 6 个字符的密码
3. 点击 "注册"

**预期结果**: 显示错误提示 "密码至少需要 6 个字符"

## 数据库验证

### 查看 Neon 数据库中的用户

1. 登录 Neon 控制台
2. 打开 SQL 编辑器
3. 执行以下查询：

```sql
SELECT id, username, "createdAt", "updatedAt" FROM "User";
```

**预期结果**: 可以看到注册的用户记录（密码已加密）

### 验证密码加密

```sql
SELECT username, password FROM "User" WHERE username = 'testuser';
```

**预期结果**: 密码字段显示的是 bcrypt 哈希值（以 `$2a$` 或 `$2b$` 开头），而不是明文

## 技术实现细节

### API 端点

- `POST /api/auth/register` - 用户注册
- `POST /api/auth/login` - 用户登录

### 数据存储

- **数据库**: Neon PostgreSQL
- **表名**: `User`
- **密码加密**: bcrypt (salt rounds = 10)
- **本地存储**: localStorage (`currentUser` 键)

### 状态管理

使用 Zustand store (`lib/userStore.ts`):
- `user`: 当前用户信息
- `isLoggedIn`: 登录状态
- `login()`: 登录方法
- `logout()`: 登出方法
- `initializeFromStorage()`: 从 localStorage 恢复状态

## 故障排除

### 问题: 注册/登录失败

**可能原因**:
1. 数据库连接失败
2. User 表未创建

**解决方案**:
1. 检查 `.env` 文件中的 `DATABASE_URL` 配置
2. 确认在 Neon 数据库中执行了 `create-user-table.sql`

### 问题: 刷新后登录状态丢失

**可能原因**: localStorage 被清除

**解决方案**: 检查浏览器控制台是否有错误，确认 `initializeFromStorage()` 被调用

### 问题: 密码验证失败

**可能原因**: bcrypt 版本不兼容

**解决方案**: 确认 `bcryptjs` 包已正确安装
