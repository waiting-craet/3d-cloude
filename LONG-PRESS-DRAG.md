# 长按拖动 + 点击聚焦功能

## ✅ 新的交互方式

### 🖱️ 点击节点（快速点击）
**操作**: 快速点击节点（按下后立即松开）

**效果**:
- ✅ 相机平滑移动，将节点移到屏幕中心
- ✅ 相机聚焦到节点
- ✅ 右侧弹出详情面板
- ✅ 顶部显示"已选中"提示

### 🖐️ 长按拖动节点
**操作**: 按住节点 300ms 后移动鼠标

**效果**:
- ✅ 进入拖拽模式
- ✅ 光标变为"抓取中"图标
- ✅ **相机完全固定**（不旋转、不移动）
- ✅ 节点跟随鼠标移动
- ✅ 松开鼠标后相机控制恢复

## 🎯 详细交互流程

### 场景 1: 点击节点查看详情
```
1. 鼠标移到节点上 → 光标变为"抓手"
2. 快速点击（< 300ms）
3. 相机开始移动
4. 节点移到屏幕中心
5. 右侧弹出详情面板
```

### 场景 2: 长按拖动节点
```
1. 鼠标移到节点上 → 光标变为"抓手"
2. 按住鼠标不放
3. 等待 300ms
4. 进入拖拽模式 → 光标变为"抓取中"
5. 相机固定（OrbitControls 禁用）
6. 移动鼠标 → 节点跟随
7. 松开鼠标 → 节点固定，相机控制恢复
```

### 场景 3: 误操作保护
```
如果按下后立即移动鼠标（< 300ms）：
- 不会进入拖拽模式
- 不会触发点击
- 相机保持正常控制
```

## 🔧 技术实现

### 长按检测
```typescript
// 按下时启动定时器
pressTimer.current = setTimeout(() => {
  setIsDragging(true)  // 300ms 后进入拖拽模式
  document.body.style.cursor = 'grabbing'
}, 300)

// 松开时清除定时器
if (pressTimer.current) {
  clearTimeout(pressTimer.current)
}
```

### 相机固定
```typescript
// KnowledgeGraph 组件监听 isDragging 状态
useEffect(() => {
  if (controlsRef.current) {
    controlsRef.current.enabled = !isDragging  // 拖拽时禁用
  }
}, [isDragging])
```

### 点击 vs 拖拽判断
```typescript
// 只有在没有移动且不是拖拽模式时才触发点击
if (!isDragging && !hasMoved.current) {
  onClick(node, e)  // 触发点击事件
}
```

## 📊 时间线

### 点击操作（< 300ms）
```
0ms    - 按下鼠标

       → 触发点击事件
       → 相机开始移动
```

### 长按拖动（> 300ms）
```
0ms    - 按下鼠标

       - 相机固定

       - 节点跟随
500ms  - 松开鼠标
       - 相机控制恢复
```

## 📝 关键参数

### 长按时间
```typescript
const LONG_PRESS_DELAY = 300  // 毫秒
```
- 太短：容易误触发拖拽
- 太长：等待时间过长
- **300ms 是最佳平衡点**

### 移动阈值
```typescript
const MOVE_THRESHOLD = 0.01  // 单位
```
- 用于判断是否真的移动了
- 避免微小抖动被识别为移动

### 相机距离
```typescript
const CAMERA_DISTANCE = 15  // 单位
```
- 点击节点后相机距离节点的距离
- 太近：看不清周围
- 太远：节点太小
