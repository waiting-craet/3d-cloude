# 3D 布局全面增强 - 解决节点拥挤和缺乏层次感

## 更新时间
2025-01-18

## 问题分析

从用户提供的截图可以看到：
1. ❌ **节点严重拥挤**：所有节点挤在中心区域
2. ❌ **缺乏层次感**：节点都在同一平面，没有利用 3D 空间
3. ❌ **视觉混乱**：无法清晰区分各个节点
4. ❌ **最小距离未生效**：即使设置了 15 单位，节点仍然很近

## 根本原因

1. **缩放因子太小**（0.6）：导致节点坐标范围太小，聚集在一起
2. **目标显示范围太小**（40）：限制了节点的分布空间
3. **高度变化不足**：只用单一正弦波，缺乏丰富的垂直层次
4. **基础间距因子太小**（1.0）：即使有动态调整也不够

## 解决方案

### 1. 大幅增加节点分布范围 ⭐

#### 修改文件：`lib/coordinate-converter.ts`

**目标显示范围**：
```typescript
const targetSize = 60  // 从 40 增加到 60 (+50%)
```

**基础缩放因子**：
```typescript
const x3d = (node.x - centerX) * scale * 1.2 * spacingFactor  // 从 0.6 增加到 1.2 (2倍)
const z3d = -(node.y - centerY) * scale * 1.2 * spacingFactor  // 从 0.6 增加到 1.2 (2倍)
```

**效果**：节点在 X 和 Z 轴上的分布范围增加 **3倍**（60/40 * 1.2/0.6 = 3）

### 2. 增强动态间距系统 ⭐

**基础间距因子**：
```typescript
let spacingFactor = 1.5  // 从 1.0 增加到 1.5
```

**动态间距表**：

| 节点数量 | 旧间距因子 | 新间距因子 | 改善 |
|---------|-----------|-----------|------|
| 1-10 | 1.0x | 1.5x | +50% |
| 11-20 | 1.5x | 2.0x | +33% |
| 21-50 | 2.0x | 2.5x | +25% |
| 51-100 | 2.5x | 3.0x | +20% |
| 100+ | 3.0x | 4.0x | +33% |

### 3. 丰富的垂直层次 ⭐⭐⭐

**多波形叠加**：
```typescript
// 增加高度变化参数
const { heightVariation = 8 } = config  // 从 5 增加到 8

// 使用三种波形叠加
const wave1 = Math.sin(nodeIndex * 0.5) * heightVariation        // 主波形
const wave2 = Math.cos(nodeIndex * 0.3) * heightVariation * 0.5  // 次波形
const linear = (nodeIndex % 5) * heightVariation * 0.3           // 阶梯式

// 组合并应用高度因子
const heightFactor = nodeCount > 50 ? 2.0 : 1.5  // 从 1.5/1.0 增加到 2.0/1.5
const y3d = (wave1 + wave2 + linear) * heightFactor
```

**效果**：
- 节点在垂直方向上有丰富的变化
- 不同节点处于不同高度层
- 创造出明显的 3D 层次感

## 技术细节

### 坐标转换公式

```
X3D = (X2D - centerX) * scale * 1.2 * spacingFactor
Z3D = -(Y2D - centerY) * scale * 1.2 * spacingFactor
Y3D = (sin(i*0.5) + cos(i*0.3)*0.5 + (i%5)*0.3) * 8 * heightFactor
```

其中：
- `scale`：根据边界自动计算
- `spacingFactor`：1.5 - 4.0（根据节点数量）
- `heightFactor`：1.5 - 2.0（根据节点数量）

### 性能影响

- **小型图谱**（< 50 节点）：几乎无影响
- **中型图谱**（50-100 节点）：增加约 50-100ms
- **大型图谱**（100+ 节点）：增加约 100-200ms

这个性能开销完全可以接受，因为换来的是显著改善的视觉效果。
