# AI预览模态框UI简化

## 更改概述
根据数据结构简化的要求，更新了AIPreviewModal组件的UI，移除了节点的type和properties字段，移除了边的properties字段。

## 具体更改

### 1. NodeEditor（节点编辑器）
**移除的字段：**
- 节点类型（type）输入框
- 属性（properties）显示区域

**新增的字段：**
- 描述（description）文本域 - 多行输入，可选

**更新后的字段：**
- 节点名称（name）- 必填
- 描述（description）- 可选，支持多行文本

### 2. EdgeEditor（边编辑器）
**移除的字段：**
- 属性（properties）显示区域

**保留的字段：**
- 连接关系显示（源节点 → 目标节点）
- 关系类型（label）- 必填，标签从"边标签"改为"关系类型"

### 3. CreateNodeModal（创建节点模态框）
**移除的字段：**
- 节点类型（type）输入框

**新增的字段：**
- 描述（description）文本域 - 多行输入，可选

**验证逻辑更新：**
- 之前：需要name和type都填写才能创建
- 现在：只需要name填写即可创建

### 4. 节点列表显示
**更新前：**
```
节点名称
类型 • 重复 • 冲突
```

**更新后：**
```
节点名称
描述（前50字符）• 重复 • 冲突
```

### 5. 冲突项显示
**更新前：**
```
节点名称
类型: xxx • 与现有节点冲突 • N个属性冲突
```

**更新后：**
```
节点名称
描述 • 与现有节点冲突 • N个属性冲突
```

## 用户体验改进
1. **更简洁**：移除了不必要的type和properties字段，界面更清爽
2. **更直观**：description使用多行文本框，方便输入较长的描述
3. **更灵活**：description是可选的，用户可以选择是否添加
4. **更友好**：关系类型输入框添加了提示文本，帮助用户理解

## 向后兼容
- 组件仍然支持type和properties字段（可选）
- 旧数据可以正常显示和编辑
- 新创建的节点不会包含type和properties

4. 查看节点列表，验证显示description而不是type
