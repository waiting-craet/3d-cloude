# 3D 节点间距和标签优化

## 修改时间
2025-01-18

## 问题描述
1. 3D 知识图谱中节点球之间的距离太近
2. 节点球没有显示名称标签

## 解决方案

### 1. 增加节点间距

#### 修改文件：`lib/coordinate-converter.ts`
- **缩放因子调整**：从 `0.3` 增加到 `0.6`
- 这使得节点在 3D 空间中的分布更加分散
- 节点之间的距离增加了一倍

```typescript
// 修改前
const x3d = (node.x - centerX) * scale * 0.3
const z3d = -(node.y - centerY) * scale * 0.3

// 修改后
const x3d = (node.x - centerX) * scale * 0.6
const z3d = -(node.y - centerY) * scale * 0.6
```

#### 修改文件：`app/api/graphs/[id]/sync/route.ts` 和 `app/api/convert/route.ts`
- **最小节点距离**：从 `2` 增加到 `4`
- 这确保即使节点很近，也会被强制推开到至少 4 个单位的距离

```typescript
// 修改前
minNodeDistance: 2

// 修改后
minNodeDistance: 4
```

### 2. 优化节点标签显示

#### 修改文件：`components/GraphNodes.tsx`
优化了文本组件的参数以提高可见性：

| 参数 | 修改前 | 修改后 | 说明 |
|------|--------|--------|------|
| `position[1]` | `(node.size || 1.5) + 0.8` | `(node.size || 1.5) + 1.2` | 标签位置更高，避免与节点重叠 |
| `fontSize` | `0.7` | `0.9` | 字体更大，更易读 |
| `outlineWidth` | `0.1` | `0.15` | 描边更粗，提高对比度 |
| `outlineOpacity` | `0.9` | `1` | 描边完全不透明 |
| `maxWidth` | `6` | `8` | 允许更长的文本 |
| `depthOffset` | 无 | `-1` | 确保文本始终在前景 |

## 效果

### 节点间距
- ✅ 节点之间的距离增加了 2 倍
- ✅ 更容易区分和选择单个节点
- ✅ 减少了节点重叠的情况
- ✅ 3D 图谱看起来更加清晰和专业

### 节点标签
- ✅ 标签文字更大更清晰
- ✅ 黑色描边提供了更好的对比度
- ✅ 标签位置更高，不会被节点遮挡
- ✅ 支持更长的节点名称

## 使用说明

1. **现有数据**：刷新页面即可看到优化效果
2. **新数据**：从 2D 工作流转换时会自动应用新的间距配置
3. **同步数据**：同步操作会使用新的间距参数

## 技术细节

### 坐标转换逻辑
```
原始 2D 坐标 → 居中 → 缩放 (scale * 0.6) → 3D 坐标
```

### 最小距离强制
使用迭代算法推开距离过近的节点：
- 最大迭代次数：10 次
- 每次迭代检查所有节点对
- 如果距离 < minDistance，沿连接向量推开

### Billboard 文本
- 文本始终面向摄像机
- 使用平滑旋转避免抖动
- 只在摄像机移动超过阈值时更新

## 相关文件
