# 3D 节点间距和标签优化

## 修改时间
2025-01-18

## 更新历史
- **第一次优化**：增加基础间距（缩放因子 0.3 → 0.6，最小距离 2 → 4）
- **第二次优化**：智能动态间距 + 更大的最小距离（最小距离 4 → 6）

## 问题描述
1. 3D 知识图谱中节点球之间的距离太近
2. 节点球没有显示名称标签
3. **节点数量很多时显得非常拥挤**

## 解决方案

### 1. 智能动态间距系统 ⭐ NEW

#### 修改文件：`lib/coordinate-converter.ts`
实现了根据节点数量自动调整间距的智能算法：

```typescript
// 动态间距因子
if (nodeCount <= 10) {
  spacingFactor = 1.0      // 10个节点以内：正常间距
} else if (nodeCount <= 20) {
  spacingFactor = 1.5      // 11-20个节点：1.5倍间距
} else if (nodeCount <= 50) {
  spacingFactor = 2.0      // 21-50个节点：2倍间距
} else if (nodeCount <= 100) {
  spacingFactor = 2.5      // 51-100个节点：2.5倍间距
} else {
  spacingFactor = 3.0      // 100+个节点：3倍间距
}

// 应用到坐标转换
const x3d = (node.x - centerX) * scale * 0.6 * spacingFactor
const z3d = -(node.y - centerY) * scale * 0.6 * spacingFactor
```

**效果**：
- 少量节点（≤10）：保持紧凑，易于查看
- 中等节点（11-50）：自动增加 1.5-2 倍间距
- 大量节点（51-100）：自动增加 2.5 倍间距
- 超多节点（100+）：自动增加 3 倍间距

### 2. 增强的高度变化

对于节点数量超过 50 的图谱，增加垂直空间利用：

```typescript
const heightFactor = nodeCount > 50 ? 1.5 : 1.0
const y3d = Math.sin(nodeIndex * 0.5) * heightVariation * heightFactor
```

### 3. 更大的最小节点距离

#### 修改文件：`app/api/graphs/[id]/sync/route.ts` 和 `app/api/convert/route.ts`
- **最小节点距离**：从 `4` 增加到 `6`
- 确保即使在最拥挤的情况下，节点之间也有足够的空间

### 4. 优化的距离强制算法

#### 修改文件：`lib/coordinate-converter.ts`
- **迭代次数**：从 10 次增加到 20 次
- **推力因子**：添加 1.2 倍推力，使节点推得更远

```typescript
const pushDistance = (minDistance - distance) / 2 * 1.2  // 1.2倍推力
```

## 技术细节

### 智能间距算法
```
1. 统计节点数量
2. 根据数量确定间距因子（1.0x - 3.0x）
3. 应用到坐标转换：coordinate * scale * 0.6 * spacingFactor
4. 强制最小距离（6 单位）
5. 迭代推开过近的节点（20次迭代，1.2倍推力）
```

### 性能优化
- 迭代算法在 O(n²) 时间复杂度内完成
- 最多 20 次迭代，通常 5-10 次即可收敛
- 对于 100+ 节点的图谱，转换时间仍在可接受范围内
