# 快速开始 - 项目图谱系统

## 🚀 系统已就绪

你的项目-图谱-节点系统已经完全实现并部署到Neon数据库！

## 📋 系统架构

```
项目 (Project)
  └── 图谱 (Graph)
       └── 节点 (Node) + 边 (Edge)
```

每个层级都有唯一的ID，数据完全隔离，互不干扰。

## ✅ 已完成的功能

### 1. 数据库表结构 (Neon PostgreSQL)
- ✅ Project 表 - 存储项目信息
- ✅ Graph 表 - 存储图谱信息
- ✅ Node 表 - 存储节点信息（支持图片、视频URL）
- ✅ Edge 表 - 存储节点关系

### 2. 完整的 API 接口
- ✅ 项目管理 (创建、查询、删除)
- ✅ 图谱管理 (创建、查询、更新、删除)
- ✅ 节点管理 (创建、查询)
- ✅ 边管理 (创建、查询)

### 3. 数据关联机制
- ✅ 项目 → 图谱 (一对多)
- ✅ 图谱 → 节点 (一对多)
- ✅ 图谱 → 边 (一对多)
- ✅ 节点 ↔ 边 (多对多)
- ✅ 级联删除 (删除项目自动删除所有关联数据)

## 📡 API 端点总览

### 项目 API
- `POST /api/projects` - 创建项目
- `GET /api/projects` - 获取所有项目
- `GET /api/projects/{id}` - 获取项目详情
- `DELETE /api/projects/{id}` - 删除项目
- `GET /api/projects/{id}/graphs` - 获取项目的图谱
- `GET /api/projects/{id}/nodes` - 获取项目的节点
- `GET /api/projects/{id}/edges` - 获取项目的边

### 图谱 API
- `POST /api/graphs` - 创建图谱
- `GET /api/graphs?projectId={id}` - 获取图谱列表
- `GET /api/graphs/{id}` - 获取图谱详情
- `PATCH /api/graphs/{id}` - 更新图谱
- `DELETE /api/graphs/{id}` - 删除图谱
- `GET /api/graphs/{id}/nodes` - 获取图谱的节点
- `GET /api/graphs/{id}/edges` - 获取图谱的边

### 节点 API
- `POST /api/graphs/{id}/nodes` - 在图谱中创建节点
- `GET /api/graphs/{id}/nodes` - 获取图谱的节点

### 边 API
- `POST /api/graphs/{id}/edges` - 在图谱中创建边
- `GET /api/graphs/{id}/edges` - 获取图谱的边

## 🎨 前端集成示例

### 在React组件中使用

```typescript
import { useState, useEffect } from 'react'

function MyComponent() {
  const [projects, setProjects] = useState([])
  const [currentProjectId, setCurrentProjectId] = useState(null)
  const [nodes, setNodes] = useState([])
  const [edges, setEdges] = useState([])

  // 加载项目列表
  useEffect(() => {
    fetch('/api/projects')
      .then(res => res.json())
      .then(data => setProjects(data.projects))
  }, [])

  // 加载当前项目的节点和边
  useEffect(() => {
    if (!currentProjectId) return

    // 加载节点
    fetch(`/api/projects/${currentProjectId}/nodes`)
      .then(res => res.json())
      .then(data => setNodes(data.nodes))

    // 加载边
    fetch(`/api/projects/${currentProjectId}/edges`)
      .then(res => res.json())
      .then(data => setEdges(data.edges))
  }, [currentProjectId])

  // 在2D/3D视图中渲染
  return (
    <div>
      {/* 项目选择器 */}
      <select onChange={(e) => setCurrentProjectId(e.target.value)}>
        {projects.map(p => (
          <option key={p.id} value={p.id}>{p.name}</option>
        ))}
      </select>

      {/* 渲染节点和边 */}
      <Canvas>
        {nodes.map(node => (
          <Node
            key={node.id}
            position={[node.x, node.y, node.z]}
            color={node.color}
            size={node.size}
            imageUrl={node.imageUrl}
          />
        ))}
        {edges.map(edge => (
          <Edge
            key={edge.id}
            from={edge.fromNodeId}
            to={edge.toNodeId}
            color={edge.color}
          />
        ))}
      </Canvas>
    </div>
  )
}
```

## 🔑 核心特性

### 1. 数据隔离
- 每个项目的数据完全独立
- 切换项目时只显示当前项目的节点和边
- 不同项目的数据互不干扰

### 2. 媒体支持
节点支持三种媒体URL：
- `imageUrl` - 节点图片
- `iconUrl` - 节点图标
- `coverUrl` - 封面/视频

### 3. 2D/3D 兼容
- 节点存储 x, y, z 坐标
- 2D视图使用 x, y
- 3D视图使用 x, y, z
- 同一数据源，不同视图

### 4. 级联删除
- 删除项目 → 自动删除所有图谱、节点、边
- 删除图谱 → 自动删除所有节点、边
- 自动清理媒体文件

## 🎯 使用场景

### 场景1: 创建新项目
1. 用户点击"新建项目"
2. 输入项目名称和描述
3. 调用 `POST /api/projects`
4. 获得项目ID

### 场景2: 在项目中创建图谱
1. 选择一个项目
2. 点击"新建图谱"
3. 输入图谱名称和描述
4. 调用 `POST /api/projects/{projectId}/graphs`
5. 获得图谱ID

### 场景3: 在图谱中添加节点
1. 选择一个图谱
2. 在2D/3D视图中点击位置
3. 输入节点信息
4. 上传图片到Vercel Blob（可选）
5. 调用 `POST /api/graphs/{graphId}/nodes`
6. 节点出现在视图中

### 场景4: 连接节点
1. 选择起始节点
2. 选择目标节点
3. 选择关系类型
4. 调用 `POST /api/graphs/{graphId}/edges`
5. 边出现在视图中

### 场景5: 切换项目
1. 用户选择不同的项目
2. 调用 `GET /api/projects/{newProjectId}/nodes`
3. 调用 `GET /api/projects/{newProjectId}/edges`
4. 视图更新，只显示新项目的数据

## 💡 最佳实践

1. **创建顺序**: 项目 → 图谱 → 节点 → 边
2. **查询优化**: 使用项目/图谱级别的API，避免全局查询
3. **媒体文件**: 先上传到Vercel Blob，再保存URL到数据库
4. **错误处理**: 所有API调用都应该有错误处理
5. **加载状态**: 显示加载指示器，提升用户体验
