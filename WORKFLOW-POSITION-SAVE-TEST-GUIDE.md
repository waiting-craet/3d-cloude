# Workflow位置保存功能 - 测试指南

## 快速测试步骤

### 测试1：返回按钮保存位置

1. 启动开发服务器：`npm run dev`
2. 访问workflow页面
3. 创建或打开一个图谱
4. 拖拽几个节点到不同位置
5. 点击左上角的"返回"按钮
6. 观察控制台输出：应该看到"✅ 返回前已保存位置"
7. 重新打开同一个图谱
8. 验证节点位置已恢复到之前的位置

**预期结果**：
- 返回时自动保存位置
- 重新打开时位置已恢复
- 控制台显示保存成功日志

### 测试2：保存并转换为3D

1. 在workflow页面创建或打开图谱
2. 拖拽节点到不同位置
3. 点击"保存并转换为3D"按钮
4. 观察右上角的状态提示
5. 等待转换完成并跳转到3D视图
6. 返回workflow页面
7. 验证节点位置保持不变

**预期结果**：
- 转换前自动保存位置
- 显示"位置保存成功"提示
- 转换后位置数据保留

### 测试3：增量更新优化

1. 打开一个图谱
2. 拖拽一个节点
3. 点击返回按钮（触发保存）
4. 重新打开图谱
5. 不移动任何节点，直接点击返回
6. 观察控制台：应该看到"⏭️ 节点位置无变化，跳过保存"

**预期结果**：
- 位置无变化时跳过保存
- 减少不必要的数据库操作

### 测试4：错误处理

1. 关闭数据库或断开网络
2. 拖拽节点后点击返回
3. 观察错误提示
4. 验证即使保存失败，仍然可以返回

**预期结果**：
- 显示错误消息
- 不阻塞用户返回操作

### 测试5：向后兼容性

1. 打开一个旧的图谱（settings为null或不包含workflowPositions）
2. 验证图谱正常显示
3. 拖拽节点并保存
4. 重新打开验证位置已保存

**预期结果**：
- 旧图谱正常工作
- 首次保存后创建workflowPositions数据

## API测试

使用curl测试API端点：

```bash
curl -X POST http://localhost:3000/api/graphs/save-positions \
  -H "Content-Type: application/json" \
  -d '{
    "graphId": "your-graph-id",
    "nodes": [
      {"id": "node1", "x": 100, "y": 200},
      {"id": "node2", "x": 300, "y": 400}
    ],
    "metadata": {
      "scale": 1.0,
      "offset": {"x": 0, "y": 0}
    }
  }'
```

**预期响应**：
```json
{
  "success": true,
  "message": "位置保存成功"
}
```

## 常见问题

### Q: 位置没有保存？
A: 检查控制台是否有错误日志，验证用户是否有权限修改图谱

### Q: 位置没有恢复？
A: 检查settings字段是否包含workflowPositions数据，查看控制台是否有解析错误

### Q: 保存失败显示401错误？
A: 用户未登录，需要先登录

### Q: 保存失败显示403错误？
A: 用户没有权限修改该图谱

### Q: 保存失败显示404错误？
A: 图谱不存在，可能已被删除
