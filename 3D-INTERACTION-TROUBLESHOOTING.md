## 如果网站无法访问

### 1. 检查服务器状态

服务器应该正在运行在 `http://localhost:3000`

如果服务器未运行,请执行:
```bash
npm run dev
```

### 2. 检查浏览器控制台

打开浏览器开发者工具(F12),查看控制台是否有错误信息。

### 3. 常见问题

#### 问题1: 页面空白或加载失败

**可能原因**:
- 数据库连接问题
- 环境变量未配置
- 端口被占用

**解决方案**:
```bash
# 检查环境变量
cat .env

# 确保数据库连接正常
npm run prisma:generate

# 尝试使用不同端口
PORT=3001 npm run dev
```

#### 问题2: 3D场景不显示

**可能原因**:
- WebGL不支持
- Three.js加载失败
- 节点数据为空

**解决方案**:
1. 检查浏览器是否支持WebGL
2. 打开控制台查看Three.js相关错误
3. 确认数据库中有节点数据

#### 问题3: 点击节点无反应

**可能原因**:
- 事件处理器未正确绑定
- 摄像机控制器被禁用

**解决方案**:
1. 检查控制台是否有JavaScript错误
2. 确认OrbitControls未被禁用
3. 尝试刷新页面

### 4. 验证新功能

打开浏览器访问 http://localhost:3000 后:

#### 测试摄像机聚焦:
1. 点击任意节点
2. 观察摄像机是否平滑移动到节点
3. 节点应该居中显示并占据合适的屏幕空间

#### 测试Billboard效果:
1. 旋转视角(鼠标拖拽)
2. 观察节点名称是否始终面向你
3. 文字应该保持垂直,不倒置

#### 测试动画:
1. 快速连续点击不同节点
2. 动画应该平滑过渡,无卡顿
3. 距离近的节点动画时间短(600ms)
4. 距离远的节点动画时间长(1000ms)
