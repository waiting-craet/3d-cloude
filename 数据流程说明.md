# 项目-图谱-节点 数据流程说明

## 📊 数据结构层次

```
┌─────────────────────────────────────────────────────────────┐
│                         项目 (Project)                       │
│  id: "cmk123..."                                            │
│  name: "委屈"                                               │
│  description: "项目简介"                                     │
│  nodeCount: 3                                               │
│  edgeCount: 1                                               │
└─────────────────────────────────────────────────────────────┘
                              │
                              │ 包含多个图谱
                              ▼
        ┌─────────────────────────────────────────────────────┐
        │              图谱A (Graph)                          │
        │  id: "cmk456..."                                    │
        │  name: "知识图谱A"                                  │
        │  projectId: "cmk123..."  ◄── 关联到项目            │
        │  nodeCount: 2                                       │
        │  edgeCount: 1                                       │
        └─────────────────────────────────────────────────────┘
                              │
                              │ 包含多个节点和边
                              ▼
        ┌─────────────────────────────────────────────────────┐
        │           节点1 (Node)                              │
        │  id: "cmk789..."                                    │
        │  name: "核心概念1"                                  │
        │  projectId: "cmk123..."  ◄── 关联到项目            │
        │  graphId: "cmk456..."    ◄── 关联到图谱            │
        │  x: 0, y: 0, z: 0                                   │
        │  color: "#FF6B6B"                                   │
        │  imageUrl: "https://blob.../image.jpg"              │
        └─────────────────────────────────────────────────────┘
                              │
                              │ 通过边连接
                              ▼
        ┌─────────────────────────────────────────────────────┐
        │              边 (Edge)                              │
        │  id: "cmkabc..."                                    │
        │  fromNodeId: "cmk789..."  ◄── 起始节点             │
        │  toNodeId: "cmkdef..."    ◄── 目标节点             │
        │  projectId: "cmk123..."   ◄── 关联到项目           │
        │  graphId: "cmk456..."     ◄── 关联到图谱           │
        │  label: "RELATES_TO"                                │
        └─────────────────────────────────────────────────────┘
```

## 🔄 创建流程

### 步骤1: 创建项目
```
用户输入: 项目名称 "委屈"
         项目描述 "我的第一个项目"
         
API调用: POST /api/projects
         {
           "name": "委屈",
           "description": "我的第一个项目"
         }

数据库: INSERT INTO Project
        生成唯一ID: "cmk123..."

返回: {
        "project": {
          "id": "cmk123...",
          "name": "委屈",
          "description": "我的第一个项目",
          "nodeCount": 0,
          "edgeCount": 0
        }
      }
```

### 步骤2: 创建图谱
```
用户输入: 图谱名称 "知识图谱A"
         图谱描述 "核心知识图谱"
         所属项目: "cmk123..."

API调用: POST /api/projects/cmk123.../graphs
         {
           "name": "知识图谱A",
           "description": "核心知识图谱"
         }

数据库: INSERT INTO Graph
        生成唯一ID: "cmk456..."
        关联项目: projectId = "cmk123..."

返回: {
        "graph": {
          "id": "cmk456...",
          "name": "知识图谱A",
          "projectId": "cmk123...",
          "nodeCount": 0,
          "edgeCount": 0
        }
      }
```

### 步骤3: 创建节点
```
用户输入: 节点名称 "核心概念1"
         节点类型 "concept"
         位置: x=0, y=0, z=0
         颜色: "#FF6B6B"
         图片: 上传到Vercel Blob

API调用: POST /api/graphs/cmk456.../nodes
         {
           "name": "核心概念1",
           "type": "concept",
           "x": 0, "y": 0, "z": 0,
           "color": "#FF6B6B",
           "imageUrl": "https://blob.../image.jpg"
         }

数据库: INSERT INTO Node
        生成唯一ID: "cmk789..."
        关联项目: projectId = "cmk123..."
        关联图谱: graphId = "cmk456..."
        
        UPDATE Graph SET nodeCount = nodeCount + 1
        UPDATE Project SET nodeCount = nodeCount + 1

返回: {
        "node": {
          "id": "cmk789...",
          "name": "核心概念1",
          "projectId": "cmk123...",
          "graphId": "cmk456...",
          "imageUrl": "https://blob.../image.jpg"
        }
      }
```

### 步骤4: 创建边
```
用户输入: 起始节点 "cmk789..."
         目标节点 "cmkdef..."
         关系类型 "RELATES_TO"

API调用: POST /api/graphs/cmk456.../edges
         {
           "fromNodeId": "cmk789...",
           "toNodeId": "cmkdef...",
           "label": "RELATES_TO"
         }

数据库: INSERT INTO Edge
        生成唯一ID: "cmkabc..."
        关联项目: projectId = "cmk123..."
        关联图谱: graphId = "cmk456..."
        
        UPDATE Graph SET edgeCount = edgeCount + 1
        UPDATE Project SET edgeCount = edgeCount + 1

返回: {
        "edge": {
          "id": "cmkabc...",
          "fromNodeId": "cmk789...",
          "toNodeId": "cmkdef...",
          "projectId": "cmk123...",
          "graphId": "cmk456..."
        }
      }
```

## 🔍 查询流程

### 查询项目的所有数据
```
API调用: GET /api/projects/cmk123...

数据库查询:
  1. SELECT * FROM Project WHERE id = "cmk123..."
  2. SELECT * FROM Graph WHERE projectId = "cmk123..."
  3. SELECT * FROM Node WHERE projectId = "cmk123..."
  4. SELECT * FROM Edge WHERE projectId = "cmk123..."

返回: {
        "project": { ... },
        "graphs": [ ... ],
        "nodes": [ ... ],
        "edges": [ ... ]
      }
```

### 查询图谱的所有数据
```
API调用: GET /api/graphs/cmk456...

数据库查询:
  1. SELECT * FROM Graph WHERE id = "cmk456..."
  2. SELECT * FROM Node WHERE graphId = "cmk456..."
  3. SELECT * FROM Edge WHERE graphId = "cmk456..."

返回: {
        "graph": { ... },
        "nodes": [ ... ],
        "edges": [ ... ]
      }
```


## 🗑️ 删除流程（级联删除）

### 删除项目
```
API调用: DELETE /api/projects/cmk123...

数据库操作:
  1. 查询项目的所有节点，获取媒体URL
     SELECT imageUrl, iconUrl, coverUrl FROM Node
     WHERE projectId = "cmk123..."
  
  2. 删除项目（级联删除所有关联数据）
     DELETE FROM Project WHERE id = "cmk123..."
     
     自动级联删除:
     - 所有图谱 (Graph)
     - 所有节点 (Node)
     - 所有边 (Edge)
  
  3. 删除Vercel Blob中的媒体文件
     DELETE FROM Blob WHERE url IN (...)

返回: {
        "success": true,
        "deletedNodeCount": 3,
        "deletedEdgeCount": 1,
        "deletedFileCount": 2
      }
```

### 删除图谱
```
API调用: DELETE /api/graphs/cmk456...

数据库操作:
  1. 查询图谱的所有节点，获取媒体URL
  2. 删除图谱（级联删除节点和边）
  3. 删除媒体文件
  4. 更新项目统计

返回: {
        "success": true,
        "deletedNodeCount": 2,
        "deletedEdgeCount": 1
      }
```

## 🎯 数据隔离机制

### 项目级隔离
```
项目A (id: "cmk111...")
  └── 图谱A1 (projectId: "cmk111...")
       └── 节点A1-1 (projectId: "cmk111...", graphId: "...")
       └── 节点A1-2 (projectId: "cmk111...", graphId: "...")

项目B (id: "cmk222...")
  └── 图谱B1 (projectId: "cmk222...")
       └── 节点B1-1 (projectId: "cmk222...", graphId: "...")
       └── 节点B1-2 (projectId: "cmk222...", graphId: "...")

查询项目A的节点:
  SELECT * FROM Node WHERE projectId = "cmk111..."
  ✅ 只返回节点A1-1, A1-2
  ❌ 不会返回节点B1-1, B1-2
```

### 图谱级隔离
```
图谱A (id: "cmk333...")
  └── 节点A-1 (graphId: "cmk333...")
  └── 节点A-2 (graphId: "cmk333...")

图谱B (id: "cmk444...")
  └── 节点B-1 (graphId: "cmk444...")
  └── 节点B-2 (graphId: "cmk444...")

查询图谱A的节点:
  SELECT * FROM Node WHERE graphId = "cmk333..."
  ✅ 只返回节点A-1, A-2
  ❌ 不会返回节点B-1, B-2
```

## 🔐 数据一致性保证

### 双重关联
```
节点同时关联项目和图谱:
  Node {
    id: "cmk789...",
    projectId: "cmk123...",  ◄── 确保属于正确的项目
    graphId: "cmk456...",    ◄── 确保属于正确的图谱
    ...
  }

边同时关联项目和图谱:
  Edge {
    id: "cmkabc...",
    projectId: "cmk123...",  ◄── 确保属于正确的项目
    graphId: "cmk456...",    ◄── 确保属于正确的图谱
    fromNodeId: "cmk789...",
    toNodeId: "cmkdef...",
    ...
  }
```

### 外键约束
```
Graph.projectId → Project.id
  删除项目时自动删除所有图谱

Node.projectId → Project.id
Node.graphId → Graph.id
  删除项目/图谱时自动删除所有节点

Edge.projectId → Project.id
Edge.graphId → Graph.id
Edge.fromNodeId → Node.id
Edge.toNodeId → Node.id
  删除项目/图谱/节点时自动删除相关边
```

## 📱 前端使用示例

### 2D视图显示
```javascript
// 1. 获取当前项目的所有节点
const nodes = await fetch(`/api/projects/${projectId}/nodes`)
  .then(r => r.json())
  .then(d => d.nodes)

// 2. 在2D画布上渲染
nodes.forEach(node => {
  drawNode2D(
    node.x,        // X坐标
    node.y,        // Y坐标
    node.color,    // 颜色
    node.size,     // 大小
    node.name      // 名称
  )
  
  // 显示节点图片
  if (node.imageUrl) {
    drawImage(node.imageUrl, node.x, node.y)
  }
})
```

### 3D视图显示
```javascript
// 1. 获取当前图谱的所有节点
const nodes = await fetch(`/api/graphs/${graphId}/nodes`)
  .then(r => r.json())
  .then(d => d.nodes)

// 2. 在3D场景中渲染
nodes.forEach(node => {
  drawNode3D(
    node.x,        // X坐标
    node.y,        // Y坐标
    node.z,        // Z坐标
    node.color,    // 颜色
    node.size,     // 大小
    node.name      // 名称
  )
  
  // 显示节点图片
  if (node.imageUrl) {
    drawImage3D(node.imageUrl, node.x, node.y, node.z)
  }
})
```

### 切换项目
```javascript
function switchProject(newProjectId) {
  // 1. 更新当前项目ID
  currentProjectId = newProjectId
  
  // 2. 清空当前视图
  clearCanvas()
  
  // 3. 加载新项目的数据
  loadProjectData(newProjectId)
  
  // 4. 重新渲染
  renderNodes()
  renderEdges()
}
```

## 💡 关键要点

1. **唯一标识**: 每个实体都有唯一的 `id` (cuid格式)
2. **双重关联**: 节点和边同时关联项目和图谱
3. **级联删除**: 删除父级自动删除所有子级
4. **数据隔离**: 不同项目/图谱的数据完全独立
5. **媒体支持**: 节点可以存储图片、视频URL
6. **2D/3D兼容**: 同一数据源支持不同视图
