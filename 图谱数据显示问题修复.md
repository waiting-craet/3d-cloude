# 图谱数据显示问题修复

## 🔍 问题诊断

### 问题描述
选择项目和图谱后，在3D和2D视图中都无法显示数据库中已有的节点和边数据。

### 根本原因分析

#### 问题1: `fetchGraph` 函数使用错误的API
**位置**: `lib/store.ts`

**问题代码**:
```typescript
fetchGraph: async () => {
  const [nodesRes, edgesRes] = await Promise.all([
    fetch('/api/nodes'),      // ❌ 使用全局API
    fetch('/api/edges'),      // ❌ 使用全局API
  ])
}
```

**问题说明**:
- 使用的是全局API（`/api/nodes`和`/api/edges`）
- 这些API返回所有节点和边，不区分图谱
- 没有检查当前选择的图谱
- 无法实现数据隔离

#### 问题2: `KnowledgeGraph` 组件不响应图谱变化
**位置**: `components/KnowledgeGraph.tsx`

**问题代码**:
```typescript
useEffect(() => {
  fetchGraph()
}, [fetchGraph])  // ❌ 只在组件挂载时执行一次
```

**问题说明**:
- 只在组件首次挂载时加载数据
- 切换图谱时不会重新加载
- 没有监听`currentGraph`的变化

#### 问题3: `WorkflowCanvas` 组件没有数据加载逻辑
**位置**: `components/WorkflowCanvas.tsx`

**问题说明**:
- 2D视图组件完全没有从数据库加载数据的代码
- 只能手动创建新节点，无法显示已有数据
- 没有连接到store的数据源

## 🔧 修复方案

### 修复1: 更新 `fetchGraph` 函数

**修复后的代码**:
```typescript
fetchGraph: async () => {
  try {
    const state = get()
    const currentGraph = state.currentGraph
    
    // 如果没有选择图谱，清空数据
    if (!currentGraph) {
      console.log('⚠️ 没有选择图谱，清空节点和边')
      set({ nodes: [], edges: [] })
      return
    }
    
    console.log('🔄 正在加载图谱数据:', currentGraph.name, currentGraph.id)
    
    // 使用图谱专属API加载数据
    const [nodesRes, edgesRes] = await Promise.all([
      fetch(`/api/graphs/${currentGraph.id}/nodes`),  // ✅ 使用图谱API
      fetch(`/api/graphs/${currentGraph.id}/edges`),  // ✅ 使用图谱API
    ])
    
    if (nodesRes.ok && edgesRes.ok) {
      const nodesData = await nodesRes.json()
      const edgesData = await edgesRes.json()
      
      const nodes = nodesData.nodes || []
      const edges = edgesData.edges || []
      
      console.log('✅ 图谱数据加载成功:', nodes.length, '个节点,', edges.length, '条边')
      set({ nodes, edges })
    }
  } catch (error) {
    console.error('❌ 获取图谱数据失败:', error)
    set({ nodes: [], edges: [] })
  }
}
```

**改进点**:
- ✅ 检查是否选择了图谱
- ✅ 使用图谱专属API（`/api/graphs/{id}/nodes`）
- ✅ 添加详细的日志输出
- ✅ 正确处理响应数据格式（`nodesData.nodes`）
- ✅ 错误处理更完善

### 修复3: 更新 `WorkflowCanvas` 组件

**修复后的代码**:
```typescript
import { useGraphStore } from '@/lib/store'

const WorkflowCanvas = forwardRef<WorkflowCanvasRef>((props, ref) => {
  // ... 其他状态
  
  // 从store获取当前图谱
  const { currentGraph, nodes: storeNodes, edges: storeEdges } = useGraphStore()

  // 加载当前图谱的数据
  useEffect(() => {
    if (!currentGraph) {
      console.log('⚠️ 2D视图: 没有选择图谱')
      return
    }

    console.log('🔄 2D视图: 加载图谱数据:', currentGraph.name)

    // 将3D节点转换为2D节点
    const converted2DNodes: Node[] = storeNodes.map((node, index) => ({
      id: node.id,
      label: node.name,
      description: node.description || '',
      x: node.x * 50 + 300, // 转换坐标
      y: node.y * 50 + 300,
      width: 200,
      height: 100,
      isEditing: false,
      imageUrl: node.imageUrl,
      videoUrl: node.videoUrl,
      mediaType: node.imageUrl ? 'image' : node.videoUrl ? 'video' : null,
    }))

    // 将3D边转换为2D连接
    const converted2DConnections: Connection[] = storeEdges.map(edge => ({
      id: edge.id,
      from: edge.fromNodeId,
      to: edge.toNodeId,
      label: edge.label,
    }))

    console.log('✅ 2D视图: 加载了', converted2DNodes.length, '个节点和', converted2DConnections.length, '条连接')

    setNodes(converted2DNodes)
    setConnections(converted2DConnections)
  }, [currentGraph, storeNodes, storeEdges])
  
  // ... 其余代码
})
```

**改进点**:
- ✅ 连接到store的数据源
- ✅ 监听图谱和数据变化
- ✅ 自动将3D数据转换为2D格式
- ✅ 保留媒体URL信息
- ✅ 添加调试日志

## 📊 数据流程

### 完整的数据加载流程

```
1. 用户选择图谱
   ↓
2. TopNavbar调用 switchGraph(projectId, graphId)
   ↓
3. switchGraph更新currentGraph并加载数据
   ├─ fetch(`/api/graphs/${graphId}/nodes`)
   └─ fetch(`/api/graphs/${graphId}/edges`)
   ↓
4. store更新nodes和edges
   ↓
5. 组件响应数据变化
   ├─ KnowledgeGraph (3D视图)
   │   └─ useEffect监听currentGraph变化
   │       └─ 调用fetchGraph()重新加载
   │           └─ 3D场景更新
   └─ WorkflowCanvas (2D视图)
       └─ useEffect监听storeNodes和storeEdges
           └─ 转换为2D格式
               └─ 2D画布更新
```

### API调用链

```
选择图谱
  ↓
GET /api/projects/{projectId}/graphs
  ↓ (获取图谱列表)
  ↓
switchGraph(projectId, graphId)
  ↓
GET /api/graphs/{graphId}/nodes
GET /api/graphs/{graphId}/edges
  ↓ (获取节点和边)
  ↓
更新UI显示
```

## 🔍 调试技巧

### 如果3D视图仍然是空的

1. **检查是否选择了图谱**
   ```javascript
   // 在控制台执行
   useGraphStore.getState().currentGraph
   ```
   应该返回图谱对象，而不是null

2. **检查API响应**
   - 打开Network标签
   - 查找`/api/graphs/{id}/nodes`请求
   - 检查响应状态码（应该是200）
   - 检查响应数据（应该有nodes数组）

3. **检查store状态**
   ```javascript
   // 在控制台执行
   useGraphStore.getState().nodes
   useGraphStore.getState().edges
   ```
   应该返回节点和边数组

4. **清理缓存**
   ```
   访问: http://localhost:3000/clear-cache
   ```

### 如果2D视图是空的

1. **检查是否从3D视图跳转**
   - 2D视图依赖store中的数据
   - 需要先在3D视图选择图谱
   - 然后再跳转到2D视图

2. **检查数据转换**
   - 查看控制台的"2D视图: 加载了X个节点"日志
   - 确认转换逻辑正确执行
