# 需求文档

## 介绍

导入页面3D统一化功能旨在简化用户的数据导入体验，通过移除2D/3D图谱类型选择，统一使用3D模式，减少用户决策复杂性，同时确保现有2D数据的向后兼容性。

## 术语表

- **导入系统**: 处理用户数据文件上传和图谱创建的系统组件
- **坐标转换器**: 负责将2D坐标自动转换为3D坐标的服务组件
- **图谱数据**: 包含节点和边信息的结构化数据
- **模板生成器**: 生成标准格式数据模板的系统组件
- **数据验证器**: 验证导入数据格式和完整性的组件

## 需求

### 需求 1

**用户故事**: 作为用户，我希望有一个简化的导入界面，这样我就不需要在2D和3D图谱类型之间做选择。

#### 验收标准

1. 当用户访问导入页面时，导入系统应该显示不包含图谱类型选择器的简化界面
2. 当用户选择项目和图谱时，导入系统应该允许直接进行文件上传而无需指定图谱类型
3. 当用户完成导入流程时，导入系统应该自动将所有数据处理为3D格式

### 需求 2

**用户故事**: 作为用户，我希望我的现有2D数据能够自动转换为3D格式，这样我就不需要手动修改数据文件。

#### 验收标准

1. 当用户上传包含2D坐标的数据文件时，坐标转换器应该自动检测坐标系统类型
2. 当检测到2D坐标数据时，坐标转换器应该将x, y坐标转换为包含z坐标的3D格式
3. 当处理3D坐标数据时，坐标转换器应该保持原有坐标不变
4. 对于所有有效的2D坐标输入，转换后再转换应该产生相同的结果（转换幂等性）

### 需求 3

**用户故事**: 作为用户，我希望下载的模板文件是3D格式的，这样我就能了解正确的数据结构。

#### 验收标准

1. 当用户请求下载数据模板时，模板生成器应该提供包含x, y, z坐标字段的3D格式模板
2. 当生成Excel模板时，模板生成器应该包含所有必需的3D坐标列和示例数据
3. 当生成JSON模板时，模板生成器应该提供完整的3D节点和边结构示例
4. 当生成CSV模板时，模板生成器应该包含正确的3D坐标列标题

### 需求 4

**用户故事**: 作为用户，我希望系统能够验证我的导入数据，这样我就能及时发现和修复数据问题。

#### 验收标准

1. 当用户上传数据文件时，数据验证器应该检查文件格式是否受支持
2. 当验证数据结构时，数据验证器应该确保所有节点包含必需的坐标信息
3. 如果数据验证失败，那么数据验证器应该返回详细的错误信息和修复建议
4. 当数据包含重复节点ID时，数据验证器应该拒绝导入并提示用户修复
5. 当边引用不存在的节点时，数据验证器应该报告引用完整性错误

### 需求 5

**用户故事**: 作为用户，我希望导入过程是可靠的，这样我就不会丢失数据或遇到意外错误。

#### 验收标准

1. 当导入过程中发生错误时，导入系统应该提供清晰的错误消息和恢复选项
2. 当文件上传失败时，导入系统应该允许用户重新尝试上传
3. 当数据处理完成时，导入系统应该确认所有数据已成功保存到数据库
4. 当系统检测到无效坐标数据时，导入系统应该阻止导入并提供数据修复指导

### 需求 6

**用户故事**: 作为用户，我希望导入过程是高效的，这样我就不需要等待很长时间。

#### 验收标准

1. 当处理小型文件（小于1MB）时，导入系统应该在2秒内完成处理
2. 当处理大型文件时，导入系统应该显示进度指示器和预估完成时间
3. 当文件大小超过10MB时，导入系统应该拒绝上传并建议用户分割文件
4. 当处理大量节点数据时，导入系统应该使用流式处理避免内存溢出

### 需求 7

**用户故事**: 作为用户，我希望我的数据是安全的，这样我就不用担心数据泄露或恶意攻击。

#### 验收标准

1. 当用户上传文件时，导入系统应该验证文件类型和内容以防止恶意文件
2. 当处理用户输入时，导入系统应该清理和验证所有数据以防止注入攻击
3. 当用户尝试导入数据时，导入系统应该验证用户对目标项目和图谱的访问权限
4. 当存储敏感数据时，导入系统应该使用适当的加密和访问控制机制

### 需求 8

**用户故事**: 作为系统管理员，我希望系统架构是可维护的，这样我就能轻松进行更新和扩展。

#### 验收标准

1. 当修改坐标转换逻辑时，其他系统组件应该不受影响
2. 当更新数据验证规则时，用户界面和存储逻辑应该继续正常工作
3. 当添加新的文件格式支持时，现有的处理流程应该保持兼容
