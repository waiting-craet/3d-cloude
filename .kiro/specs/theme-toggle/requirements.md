# 主题切换功能需求文档

## 介绍

在三维知识图谱页面的顶部导航栏中添加一个主题切换按钮，支持亮色和暗色两种主题模式。用户可以通过点击按钮在两种主题之间切换，系统会记住用户的主题偏好。

## 术语表

- **Theme**: 应用的视觉主题，包括背景色、文本色、弹窗样式等
- **Dark Theme**: 暗色主题，三维背景保持暗色，弹窗为暗色调
- **Light Theme**: 亮色主题，三维背景为白色，弹窗为亮色调
- **TopNavbar**: 三维页面顶部的导航栏组件
- **NodeDetailPanel**: 点击节点时出现的详情弹窗
- **KnowledgeGraph**: 三维知识图谱渲染组件
- **Theme_Manager**: 负责管理和应用主题的系统

## 需求

### 需求 1: 主题切换按钮

**用户故事:** 作为用户，我想在导航栏中看到一个主题切换按钮，以便我可以在亮色和暗色主题之间切换。

#### 验收标准

1. WHEN 用户访问三维知识图谱页面 THEN TopNavbar 应该在右侧显示一个主题切换按钮
2. WHEN 用户点击主题切换按钮 THEN 系统应该在亮色和暗色主题之间切换
3. WHEN 主题切换时 THEN 按钮应该显示当前主题的视觉指示（如图标变化）
4. WHEN 用户切换主题 THEN 系统应该立即应用新主题到所有相关组件

### 需求 2: 暗色主题样式

**用户故事:** 作为用户，我想在暗色主题下有一个舒适的视觉体验，以便我可以在低光环境中使用应用。

#### 验收标准

1. WHEN 暗色主题被激活 THEN KnowledgeGraph 的三维背景应该保持暗色（不变）
2. WHEN 暗色主题被激活 THEN NodeDetailPanel 弹窗应该使用暗色调样式
3. WHEN 暗色主题被激活 THEN 弹窗文本应该使用浅色以保证可读性
4. WHEN 暗色主题被激活 THEN 弹窗边框和分隔线应该使用暗色调

### 需求 3: 亮色主题样式

**用户故事:** 作为用户，我想在亮色主题下有一个清爽的视觉体验，以便我可以在明亮环境中使用应用。

#### 验收标准

1. WHEN 亮色主题被激活 THEN KnowledgeGraph 的三维背景应该变为白色
2. WHEN 亮色主题被激活 THEN NodeDetailPanel 弹窗应该使用亮色调样式（现有样式）
3. WHEN 亮色主题被激活 THEN 弹窗文本应该使用深色以保证可读性
4. WHEN 亮色主题被激活 THEN 弹窗边框和分隔线应该使用浅色调

### 需求 4: 主题持久化

**用户故事:** 作为用户，我想让我的主题选择被记住，以便下次访问时应用会使用我上次选择的主题。

#### 验收标准

1. WHEN 用户切换主题 THEN 系统应该将主题选择保存到本地存储
2. WHEN 用户重新访问页面 THEN 系统应该恢复上次保存的主题选择
3. WHEN 本地存储中没有保存的主题 THEN 系统应该使用暗色主题作为默认主题

### 需求 5: 主题应用到所有相关组件

**用户故事:** 作为用户，我想确保主题被一致地应用到所有相关的 UI 组件，以便获得统一的视觉体验。

#### 验收标准

1. WHEN 主题改变 THEN TopNavbar 应该应用新主题的样式
2. WHEN 主题改变 THEN NodeDetailPanel 应该应用新主题的样式
3. WHEN 主题改变 THEN KnowledgeGraph 背景应该应用新主题的样式
4. WHEN 主题改变 THEN 所有其他 UI 元素应该保持视觉一致性

### 需求 6: 主题切换动画

**用户故事:** 作为用户，我想在主题切换时看到平滑的过渡效果，以便获得更好的用户体验。

#### 验收标准

1. WHEN 用户切换主题 THEN 背景颜色应该平滑过渡（不是突然改变）
2. WHEN 用户切换主题 THEN 过渡时间应该在 200-300ms 之间
3. WHEN 主题切换完成 THEN 所有元素应该完全应用新主题
