# 项目创建失败修复说明

## 问题描述

在部署环境中创建项目和图谱时，虽然创建操作成功，但在重新加载数据时找不到新创建的项目或图谱，导致以下错误：

```
❌ 创建项目失败: Error: 无法在重新加载的数据中找到新创建的项目或图谱
```

## 根本原因

这是一个**数据库读取一致性问题**，在云数据库环境（如 Neon）中常见：

1. **写入延迟**：数据写入主节点后，读取副本可能还没有同步
2. **连接池问题**：不同的 API 调用可能使用不同的数据库连接
3. **缓存问题**：浏览器或 CDN 可能缓存了旧的 API 响应

## 解决方案

### 1. 添加重试机制

在 `lib/store.ts` 的 `createProject` 和 `addGraphToProject` 函数中添加了重试逻辑：

- 最多重试 3 次
- 每次重试前等待递增的延迟时间（500ms, 1000ms, 1500ms）
- 确保数据库有足够时间完成写入和同步

### 2. 禁用缓存

在重新加载项目列表时，添加了缓存控制头：

```typescript
const projectsRes = await fetch('/api/projects/with-graphs', {
  cache: 'no-store',
  headers: {
    'Cache-Control': 'no-cache, no-store, must-revalidate',
    'Pragma': 'no-cache',
  },
})
```

### 3. 降级处理

如果重试后仍未找到数据，使用创建 API 返回的数据构建对象：

- 使用创建响应中的项目和图谱信息
- 手动构建数据结构并添加到列表中
- 确保用户界面能够正常显示

## 修改的文件

- `lib/store.ts`
  - `createProject` 函数：添加重试逻辑和降级处理
  - `addGraphToProject` 函数：添加重试逻辑和降级处理

## 测试步骤

1. 部署更新后的代码到生产环境
2. 尝试创建新项目和图谱
3. 观察控制台日志，确认：
   - ✅ 项目创建成功
   - ✅ 图谱创建成功
   - ✅ 项目列表加载成功
   - ✅ 找到新创建的项目和图谱
4. 验证新项目和图谱在界面中正确显示

## 预期结果

- 创建项目和图谱应该成功完成
- 不再出现"无法在重新加载的数据中找到"的错误
- 新创建的项目和图谱立即在界面中可见
- 如果数据库同步延迟，系统会自动重试

## 注意事项

1. 这个修复主要针对云数据库环境的一致性问题
2. 本地开发环境通常不会遇到这个问题
3. 如果问题持续存在，可能需要检查数据库配置或网络延迟
4. 重试机制会增加创建操作的总时间（最多额外 3 秒）
