# 图谱显示修复说明

## ✅ 已修复的问题

### 问题描述
在现有图谱下拉框中选择了相应的项目和图谱后，有时没有显示出相应的三维知识图谱。

### 根本原因
1. **数据加载缺失**: `switchGraph` 函数只是切换了当前图谱，但没有从数据库加载对应的节点和边数据
2. **旧数据干扰**: localStorage中存储的是本地生成的临时ID（如`graph-1768294373820`），而不是数据库的真实ID
3. **API不匹配**: 创建节点和边时使用的是旧的全局API，而不是图谱专属的API

## 🔧 修复内容

### 1. 修复 `switchGraph` 函数 (lib/store.ts)
```typescript
// 修复前：只切换状态，不加载数据
switchGraph: (projectId, graphId) => {
  set({ currentProject, currentGraph, nodes: [], edges: [] })
}

// 修复后：切换状态并从数据库加载数据
switchGraph: async (projectId, graphId) => {
  set({ currentProject, currentGraph, nodes: [], edges: [] })
  
  // 从数据库加载图谱的节点和边
  const [nodesRes, edgesRes] = await Promise.all([
    fetch(`/api/graphs/${graphId}/nodes`),
    fetch(`/api/graphs/${graphId}/edges`),
  ])
  
  const nodesData = await nodesRes.json()
  const edgesData = await edgesRes.json()
  
  set({ 
    nodes: nodesData.nodes || [], 
    edges: edgesData.edges || [] 
  })
}
```

### 2. 修复 `addNode` 和 `addEdge` 函数
```typescript
// 修复前：使用全局API
fetch('/api/nodes', { method: 'POST', ... })

// 修复后：使用图谱专属API
fetch(`/api/graphs/${currentGraph.id}/nodes`, { method: 'POST', ... })
```

### 3. 修复项目加载逻辑 (components/TopNavbar.tsx)
```typescript
// 修复前：从localStorage加载本地数据
const savedProjects = localStorage.getItem('projects')

// 修复后：从数据库加载真实数据
const res = await fetch('/api/projects')
const projects = await res.json()

// 并验证ID的有效性
const isValidId = (id: string | null) => id && id.startsWith('cmk')
```

### 4. 修复 `createProject` 和 `addGraphToProject` 函数
```typescript
// 修复前：生成本地临时ID
const projectId = `project-${Date.now()}`

// 修复后：使用数据库API创建，获取真实ID
const projectRes = await fetch('/api/projects', {
  method: 'POST',
  body: JSON.stringify({ name, description })
})
const project = await projectRes.json()
// project.id 是数据库生成的真实ID (如 cmkxxx...)
```

## 📊 数据流程

### 完整的数据加载流程
```
1. 页面加载
   ↓
2. 从数据库加载项目列表 (GET /api/projects)
   ↓
3. 为每个项目加载图谱列表 (GET /api/projects/{id}/graphs)
   ↓
4. 用户选择图谱
   ↓
5. 调用 switchGraph(projectId, graphId)
   ↓
6. 从数据库加载图谱的节点 (GET /api/graphs/{id}/nodes)
   ↓
7. 从数据库加载图谱的边 (GET /api/graphs/{id}/edges)
   ↓
8. 在3D视图中渲染节点和边
```

### 创建节点的流程
```
1. 用户在3D视图中点击添加节点
   ↓
2. 检查是否选择了图谱
   ↓
3. 调用图谱API创建节点 (POST /api/graphs/{id}/nodes)
   ↓
4. 节点自动关联到当前项目和图谱
   ↓
5. 更新本地状态，节点出现在视图中
```

## 💡 使用建议

1. **清理旧数据**: 如果之前有旧的测试数据，建议清理localStorage：
   ```javascript
   localStorage.clear()
   ```
   然后刷新页面

2. **创建新项目**: 使用"新建图谱"按钮创建项目和图谱，会自动使用数据库API

3. **数据持久化**: 所有数据都存储在Neon数据库中，刷新页面不会丢失

4. **多图谱管理**: 可以在一个项目下创建多个图谱，每个图谱的数据完全独立
