# AI图谱简化功能说明

## 已完成的修改

### 1. 简化AI生成的数据结构
- **节点**：只生成name（名称）和description（描述），不再生成type和properties
- **边**：只生成from、to和type（关系类型），不再生成properties
- **关系类型**：使用简洁的中文描述，如"属于"、"位于"、"创建"、"参与"等

### 2. 示例对比

#### 修改前
```json
{
  "entities": [
    {
      "name": "张三",
      "type": "person",
      "properties": {
        "description": "研究员",
        "age": 30
      }
    }
  ],
  "relationships": [
    {
      "from": "张三",
      "to": "北京大学",
      "type": "works_for",
      "properties": {
        "since": "2020"
      }
    }
  ]
}
```

#### 修改后
```json
{
  "entities": [
    {
      "name": "张三",
      "description": "研究员"
    }
  ],
  "relationships": [
    {
      "from": "张三",
      "to": "北京大学",
      "type": "工作于"
    }
  ]
}
```

### 3. 自动跳转功能
保存图谱后会自动跳转到3D页面，显示刚刚保存的图谱。

## 使用方法

1. 在AI文本输入页面输入文本
2. 点击"生成图谱"按钮
3. 查看预览（节点只显示名称和描述，边只显示关系类型）
4. 点击"确定保存"
5. 系统自动保存并跳转到3D页面

## 向后兼容
系统仍然支持旧格式的数据（包含type和properties），不会影响现有功能。

## 测试示例

输入文本：
```
张三在北京大学工作，他创建了一个名为AI Lab的实验室。
李四参与了这个实验室的研究，王五也在北京大学工作。
```

预期生成：
- 节点：张三、北京大学、AI Lab、李四、王五
- 边：
  - 张三 → 工作于 → 北京大学
  - 张三 → 创建 → AI Lab
  - 李四 → 参与 → AI Lab
  - 王五 → 工作于 → 北京大学
